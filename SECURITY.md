# Security Policy

## Supported Versions

- Maintain security patches for active `release/*` branches.
- `main` always receives latest fixes first.

## Reporting Vulnerabilities

Send reports to security contact (replace before use):

- Email: `security@example.com`
- Include: impact, reproduction steps, affected version, suggested mitigation.

Do not open public issues for undisclosed vulnerabilities.

## Response SLA

- Initial response: within 2 business days
- Triage decision: within 5 business days
- Patch target: based on severity and branch policy
